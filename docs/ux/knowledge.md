## Sprint 0（2026-07-04）— Initial Understanding

### Primary Persona
王凯，31岁，产品经理。每周 5-8 集播客。核心矛盾："想学 vs 记不住"。

### Primary User Flow（端到端）

```
Home
 ├─ 点 [Learn] 入口
 │   └─ Learn 页
 │       ├─ 粘贴 URL
 │       ├─ 系统抓取 → loading（可见进度）
 │       └─ 选学习目标（5 选 1）
 │           └─ 生成学习包 → loading
 │               └─ Episode 页
 │                   ├─ 学习快照卡（10 秒判断）
 │                   ├─ 3 后续动作：完整学习 / 3min速学 / 跳过
 │                   ├─ 6 步学习路径（打勾进度）
 │                   ├─ 概念解释器（点开三层解释）
 │                   ├─ 5-10 卡片（收藏/删除/编辑）
 │                   └─ 行动清单（今天/本周/长期）
 │                       └─ 勾选后进入 Review 队列
 │
 ├─ 点 [Review] 入口（复习首页）
 │   └─ 今日待复习 N 张
 │       └─ 闪卡模式（问题面 → 答案面）
 │           └─ 记得/模糊/不记得 → 更新下次复习时间
 │
 └─ 点 [Library] 入口（个人知识库）
     ├─ 卡片列表（按主题/来源/类型筛选）
     ├─ 跨集搜索
     └─ Ask AI（默认 6 个问题按钮）
```

### 关键 UX 摩擦点（初判，待 Sprint 演进验证）

1. **首次导入等待**：45 秒（抓取）+ 60 秒（GLM 生成）= 最长 105 秒。用户会不会跳出？
   - 需要**分步骤可见的 loading**（不是转圈）
   - 首屏优先展示"元数据（标题/时长/频道）"，让用户知道抓到了
2. **学习目标选择**：5 个按钮一屏显示但**不可回退**（PRD 规定）。用户误点后需重新生成 = 60 秒。
   - UX 建议：点击后 3 秒内可撤回，之后不可
3. **6 步路径进度感**：只有打勾还不够
   - 需要 header 常驻进度条（"2 / 6"），让用户随时看到位置
4. **卡片默认收藏**：PRD 定为"默认收藏"，用户可能感觉被强推
   - 需要"批量取消"入口

### UX AC 模板（用于每个 Story）

- [ ] 从入口到功能可达 ≤ 3 次操作（`TECH_SPEC §ux-thresholds`）
- [ ] 用户任何等待 > 2s 必须有可见反馈（TECH_SPEC feedback threshold）
- [ ] 完成动作后有明确正反馈（打勾 / toast / 状态变化）
- [ ] 错误状态说明清楚"发生什么 + 怎么办"
- [ ] 移动 375px 触摸目标 ≥ 44px 高

### 视觉验证（风格 F）
- 每页应有 1 个主视觉插画（不允许"全文字"页面）
- 大标题必须用 Bagel Fat One 胖字母
- 织布条作分隔（不用普通 border）
- 状态标签用白色椭圆气泡（不用矩形 chip）
