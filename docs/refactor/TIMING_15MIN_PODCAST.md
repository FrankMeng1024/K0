## 四、精学阶段 (260s, 失败) — Bug4 根因

| 子步骤 | 耗时 | tokens | 结果 |
|---|---|---|---|
| GLM 精学生成 (deep) 第1次 | 116.2s | 输入1969 / 输出**4640** | HTTP 200 但 JSON 不完整 |
| GLM 精学生成 (deep.retry) 重试 | 142.8s | 输入1969 / 输出**6663** | HTTP 200, `GLM_MALFORMED_JSON` |
| **合计** | **259s** | | ❌ 两次都解析失败 → job failed |

**Bug4 根因 (已确认)**:
- 错误码 `GLM_MALFORMED_JSON: Expected ',' or '}' after property value`
- 精学 prompt 要求生成大量结构化内容 (第1次 4640 tok, 重试 6663 tok 输出)。**输出 6663 逼近 8192 上限**, GLM 在长 JSON 生成中**截断/格式错乱**, 返回不完整 JSON。
- 不是网络/超时/额度问题 (两次都是 HTTP 200)。**是 GLM 长 JSON 输出可靠性问题**。
- 现有 retry 逻辑无效: 重试还是同 prompt 同参数, 输出更长反而更容易坏。

**修复方向 (Bug4)**:
1. 精学 prompt 拆分/瘦身, 降低单次输出长度 (远离 8192 上限)
2. 或分多次生成 (每张卡片/每步独立调用) 再拼装
3. 或加 JSON 修复兜底 (截断补全) —— 治标
4. `GLM_MAX_TOKENS` 已顶格 8192, 不能再提 (模型硬上限)
