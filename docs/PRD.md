# PRD.md — K0 播客学习操作系统

**产品代号**: K0 (Podcast Learning OS)
**版本**: v1.0 (锁定，不可直接编辑，所有变更走 CR)
**日期**: 2026-07-02

---

## 一、产品定义

**一个以播客为输入、以学习成果为输出的 AI 学习系统。**

用户只需粘贴播客链接（YouTube / Apple / Spotify），系统自动生成：
- 学习路径
- 知识卡片
- 测验题
- 行动清单
- 复习计划

帮助用户把碎片化播客内容转化为可掌握、可复习、可应用的知识。

**核心承诺**：用户导入一集播客后，能在 10-15 分钟内完成一次高质量学习，并且在一周后仍能记得核心内容。

---

## 二、目标用户

- **主要**：每周听 5+ 集播客、有信息焦虑但缺乏沉淀的知识工作者（产品经理、创业者、研究者）
- **次要**：中英双语学习者，需要跨语言内容处理

---

├─────────────────────────┤
│  📥 Learn（导入学习）    │
│  🔄 Review（每日复习）   │

## 四、Epic 拆分

### E-001: Import & Transcribe（播客导入 + 转录）
把用户提供的链接变成可学习的结构化数据。

### E-002: Learning Snapshot（学习快照）
生成 10 秒判断卡：一句话 + 3 观点 + 价值分 + 成本估。

### E-003: Learning Pack Generator（学习包生成器）
6 步学习路径 + 概念解释器 + 知识卡片 + 行动清单。

### E-004: Learning Player（学习播放器 UI）
单集页面：学习状态、快照、路径、卡片、Ask AI。

### E-005: Review System（复习系统）
测验、闪卡、复习队列（明天/三天/一周）、每日提醒。

### E-006: Knowledge Library（个人知识库）
按主题 / 来源 / 类型分类查看，跨集搜索，AI 问答。

### E-007: Foundation（基础设施）
用户体系接口预留、数据模型、鉴权中间件、部署。

## 五、Must-Have Features（MVP 承诺）

### M1. Podcast Import 播客导入
**用户价值**: 用户不需要理解转录、字幕、时间戳这些概念，粘贴链接就够了。

**功能**:
- 支持输入形式：
  - YouTube 视频链接（含播客类频道，如 Lex Fridman / All-In / 硅谷 101）
  - Apple Podcasts 单集链接
  - Spotify 单集链接
- 系统自动完成：
  - 元数据抓取（标题、嘉宾、频道、时长、发布日期、封面图）
  - 字幕/文本获取（优先官方字幕，其次自动语音转录）
  - 语言自动识别（中/英）
  - 时间戳对齐
- 导入完成后展示"导入成功"卡片，进入学习目标选择

**成功标准**:
- 一个典型 YouTube 播客链接从粘贴到"进入学习目标选择"步骤 < 60 秒（有字幕时）
- 支持中文和英文播客
- 元数据展示至少包含：标题、频道/嘉宾、时长

### M2. Learning Goal Selection 学习目标选择
**用户价值**: 让 AI 知道"我为什么学"，产出才有针对性。

**功能**:
- 导入后展示 5 个学习目标按钮（一屏显示）：
  1. **快速了解** — 我只想知道这集讲什么
  2. **深度学习** — 我想真正掌握这集内容
  3. **找可执行方法** — 我想提炼行动建议
  4. **批判性思考** — 我想判断观点是否可靠
  5. **为工作/研究使用** — 我要沉淀成可引用材料
- 用户单选，点击后不可回退（回退需重新生成）
- 用户选择决定后续所有 AI 输出的重点

**成功标准**:
- 5 个按钮清晰可见，无需滚动
- 移动端触摸友好，按钮 ≥ 44px 高度
- 点击后立即进入学习包生成 loading

### M3. Learning Snapshot 学习快照
**用户价值**: 10 秒判断这集值不值得学，学多深。

**功能**:
- 学习包生成后，第一屏展示"学习快照"卡：
  - **这集一句话**：≤ 25 字
  - **核心观点**：3 个，每个带时间戳
  - **适合谁学**：如 "产品经理 / 创业者 / 学生"
  - **学习价值评分**：3 个维度评分（信息密度 / 新观点 / 可行动性），各 1-10 分
  - **学习成本**：预估学完需要几分钟（3 / 10 / 30）
  - **最值得学的 3 个片段**：每个带时间戳 + 一句话说明
  - **可以跳过的部分**：广告 / 寒暄 / 重复（带时间戳）
- 快照上有 3 个后续动作按钮：
  - "3 分钟速学"（只看快照 + 核心观点）
  - "完整学习"（进入 6 步学习路径）
  - "跳过这集"（不生成学习包，回首页）

**成功标准**:
- 快照信息完整且真实反映内容
- 一屏内展示（不折叠核心信息）
- 中英文播客都能生成对应语言的快照

### M4. Learning Pack 学习包
**用户价值**: 不是"看摘要"，而是"完成一次学习"。

**功能 A: 6 步引导式学习路径**
每集自动生成 6 个学习步骤，每步展示核心内容 + 完成打勾：
1. **背景理解**：这集在讨论什么问题，为什么现在讨论
2. **核心观点**：嘉宾的关键判断（3-5 条）
3. **案例与证据**：嘉宾举了哪些例子、数据、故事
4. **方法论提炼**：可复用的做事方法（如果有）
5. **批判性思考**：可能的逻辑漏洞、隐含前提、反方观点
6. **我的应用**：这集对我的工作/研究有什么用（AI 建议 + 用户自选）

**功能 B: 概念解释器**
- 系统自动识别：关键概念、行业术语、人物/公司/事件、方法框架
- 每个概念可点开，三层解释：
  - **小白解释**：用简单语言讲明白
  - **原文语境**：嘉宾为什么提它（带时间戳）
  - **延伸理解**：与其他概念的关系

**功能 C: 知识卡片自动生成**
- 每集 5-10 张卡片，5 种类型：
  - **观点卡**：嘉宾的核心判断
  - **方法卡**：可复用的做事方法
  - **案例卡**：真实案例/故事
  - **反思卡**：启发性问题
  - **行动卡**：可执行建议
- 每张卡片 5 个字段：
  - 标题（一个清晰观点）
  - 类型（上述 5 类）
  - 解释（用自己的话讲明白）
  - 来源（播客名 + 时间戳，可跳转）
  - 我的应用（AI 建议 + 可编辑）
- 用户对每张卡片有 3 个动作：收藏 / 删除 / 保持不变（默认收藏）

**功能 D: 行动清单**
- 学习包结束展示 3 条行动：
  - **今天可以做的 1 件事**
  - **本周可以尝试的 1 件事**
  - **值得继续研究的 1 个问题**
- 用户可勾选，勾选后进入 Review 系统跟踪

**成功标准**:
- 学习路径清晰，用户能感觉到"完成感"（打勾进度可见）
- 卡片信息密度合理，不冗余
- 中英文播客输出对应语言

### M5. Review System 复习系统
**用户价值**: 从"看完"到"记住"。

**功能**:
- **测验题生成**：每集自动生成 3-5 道选择题 / 简答题
  - 覆盖核心观点
  - 用户答题后展示正确答案 + 来源时间戳
- **闪卡模式**：卡片正面是问题，背面是答案 + 时间戳
- **复习提醒**：每张收藏的卡片自动进入队列：
  - 明天（+1 天）
  - 三天后（+3 天）
  - 一周后（+7 天）
- **每日复习队列**：首页 Review 入口显示"今天有 N 张卡片待复习"
- **复习反馈**：用户对每张卡片选择"记得 / 模糊 / 不记得"
  - 记得：延长下次间隔
  - 模糊：保持当前节奏
  - 不记得：明天再来
- MVP 阶段只做站内展示，不做邮件/微信推送

**成功标准**:
- Review 入口点开即可开始复习，无配置
- 每张卡片的复习完成 ≤ 30 秒
- 复习状态持久化（刷新页面/换设备继续可复习）

### M6. Home Page — 3 Entry Points 首页 3 入口
**用户价值**: 极简，一眼看懂系统能帮我做什么。

**功能**:
- **Learn（导入学习）**：粘贴链接的主流程入口
- **Review（每日复习）**：显示今日待复习数量、点开进入复习流
- **Library（个人知识库）**：查看长期沉淀，按主题/来源/类型筛选，可跨集搜索
- 首页无其他复杂功能：没有推送、订阅、社区

**成功标准**:
- 首页加载 < 2 秒
- 3 个入口一屏可见（移动/桌面）
- 每个入口的功能一句话说清楚

### M7. Foundation — User System (Architecture Only) 用户体系（架构层）
**用户价值**: 第一版单用户体验简单，但未来可无缝升级多用户。

**功能**:
- 数据库所有业务表预留 `user_id` 字段（非空，MVP 阶段默认写入固定 ID = 1）
- 后端 API 接受可选 `Authorization: Bearer <token>` header
- MVP 阶段登录/注册 UI **不展示**，登录接口保留但不启用
- 未来一次 CR 就可以：解除接口开关 + 加登录 UI + 数据迁移

**成功标准**:
- 数据库 schema 支持多用户（不做迁移改造）
- API 层 auth 中间件已实现（默认放行，配置可开启）
- Frontend 状态管理预留 user 概念

## 六、非功能性要求

### 性能
- 首屏 < 2 秒
- API 响应（非 AI 生成）< 500ms
- AI 学习包生成 < 60 秒（一次全跑：快照 + 学习包 + 卡片 + 测验 + 行动）
- 转录抓取 < 45 秒（YouTube 有字幕场景）

### 视觉质量
- 遵循 `C:\ClaudeCodeProjects\FRONTEND_STANDARDS.md` 最高标准
- Product Soul: **专注、可完成、值得信赖**（不是酷炫，不是过度动画）
- 视觉隐喻: 学习卡片是"可以完成的任务"，不是"堆积的信息"
- 交互故事: 用户每完成一步都能看见进度和反馈

### 兼容性
- 移动 375px 到桌面 1440px 全响应式
- 主流浏览器（Chrome / Safari / Edge 最新两版）
- 深色模式 MVP 不做

### 数据可靠性
- 用户导入的播客、生成的学习包、收藏的卡片、复习进度：所有数据入 MySQL
- 数据库连接失败必须明确提示，不能静默失败

### 语言
- 系统 UI 使用中文（主要用户在中国）
- 学习包输出语言 = 播客源语言（中文播客出中文，英文播客出英文）

## 七、边界（同 DISCOVERY §Won't Have）

明确不做，任何 Sprint 都不加：
- 播客播放器（不做泛播客 App）
- 音频文件上传（MVP 阶段不做 STT）
- 社交/分享/评论
- 复杂知识图谱
- 手动富文本笔记编辑器
- 多人协作
- 微信/邮件推送
- 播客订阅

## 八、Success Metrics 成功指标

- **激活**：用户导入第一集播客并完成一次 3+ 分钟学习的比例 ≥ 60%
- **留存**：一周内回访使用 Review 的比例 ≥ 30%
- **完成**：学习包生成成功率 ≥ 95%（导入无问题的前提下）
- **质量**：Virtual User 最终验收评分 ≥ 9.5/10
