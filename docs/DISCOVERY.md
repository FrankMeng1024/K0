## 一句话产品定义

**把一集播客变成一次可完成、可复习、可积累、可应用的学习。**

不是"帮你总结播客"，而是"让你真正学会里面有价值的东西"。

## Competitive Analysis 竞品分析

| 产品 | 定位 | 我们的差异 |
|------|------|------------|
| **Snipd** | 播客高光片段收集 | 他们做"片段收藏"，我们做"学习闭环"（快照 → 学习路径 → 卡片 → 测验 → 复习） |
| **Podnotes / Podcastle** | AI 播客转录 + 摘要 | 他们止步于摘要，我们把内容重组为"学习任务"，用户能真正学会 |
| **NotebookLM (Google)** | 多源资料对话 + 生成音频 | 他们做"资料库对话"，我们做"单集深度学习+长期沉淀" |
| **Readwise Reader** | 阅读高亮沉淀 | 他们做书/文章，我们专攻播客这一低结构化、高信息密度的场景 |
| **飞书妙记 / 通义听悟** | 转录 + AI 总结 | 他们是"办公场景转录工具"，我们是"个人学习陪伴系统" |

**核心差异化**：
1. **学前判断卡**（10 秒决定值不值得学）—— 竞品都是先转录后总结，我们先给"预览"
2. **引导式学习路径**（6 步结构化）—— 竞品输出扁平摘要，我们输出可以打勾完成的课程
3. **批判性学习模式**（Critic Mode）—— 主动质疑嘉宾观点，竞品都是顺从式总结
4. **复习队列**（明天/三天后/一周后）—— 从"工具"变"陪伴系统"

## User Persona 用户画像

### Primary Persona: 高信息饥饿的知识工作者

**姓名**: 王凯，31 岁，产品经理

**背景**:
- 每周听 5-8 集播客（AI/创业/产品）
- 通勤时听、健身时听，但听完常常"什么都没记住"
- 会截图保存高光片段，但从不回看
- 试过用 Notion 整理，但整理成本太高，放弃

**痛点**:
1. **信息焦虑**：怕错过好观点，但没有时间深听
2. **无沉淀**：听完就忘，无法复用到工作
3. **难判断**：一集 90 分钟的播客不知道值不值得深听
4. **无批判**：容易被嘉宾观点带走，缺乏独立判断

**期待**:
- 10 秒判断这集值不值得学
- 3-15 分钟完成一次"高质量学习"
- 有系统帮我记住、复习、应用
- 卡片能沉淀成个人知识库

### Secondary Persona: 中英双语的研究/学习者

- 一半英文播客（Lex Fridman, All-In, Acquired, a16z）
- 一半中文播客（面基, 硅谷 101, 张小珺访谈）
- 需要中英文都能自然处理
- 希望学习包输出语言与原播客一致

## Feature Priority — MoSCoW

### Must Have（MVP 必须做）

**M1. 播客导入**
- YouTube 链接导入（自动抓字幕）
- Apple Podcasts / Spotify 播客链接（RSS 解析 + 音频转录）
- 自动识别：标题、嘉宾、时长、语言、转录文本 + 时间戳

**M2. 学习目标选择**（5 选 1）
- 快速了解 / 深度学习 / 找可执行方法 / 批判性思考 / 为工作研究使用
- 决定后续输出的重点

**M3. 学习快照 Learning Snapshot**
- 一句话总结
- 3 个核心观点（带时间戳）
- 学习价值评分（信息密度 / 新观点 / 可行动性）
- 学习成本（3/10/30 分钟）
- 最值得学的 3 个点 + 可跳过的部分

**M4. 学习包 Learning Pack**
- 6 步引导式学习路径（背景 → 核心观点 → 案例 → 方法论 → 批判 → 我的应用）
- 概念解释器（关键术语三层解释）
- 知识卡片（观点/方法/案例/反思/行动 5 类，每张 5 字段）
- 行动清单（今天/本周/长期各 1 件）

**M5. 复习系统**
- 收藏卡片
- 测验题生成（每集 3-5 题）
- 复习提醒（明天 / 三天后 / 一周后）
- 每日复习队列

**M6. 首页 3 入口**
- Learn（导入新播客）
- Review（每日复习队列）
- Library（个人知识库，按主题/来源/类型查看）

**M7. 用户体系（架构层）**
- 数据库预留 `user_id` 字段
- 第一版 UI 隐藏登录/注册，默认单用户
- 后端接口保留 Bearer token 参数（可空）
- 后续可无缝升级为多用户

### Should Have（后续 Sprint 可能做）

- 单集详情页 Ask AI（默认问题按钮）
- 批判模式 Critic Mode 独立入口
- 主题跨集聚合（"我关于 AI Agent 学了什么"）
- 学习进度打勾 / 完成感反馈

### Could Have（可有可无）

- 音频文件直接上传（需 STT，成本高，MVP 不做）
- 深色模式
- 播客搜索/订阅（会稀释定位）

### Won't Have — What We Will NOT Build

明确不做，写死不允许中途加：

| 不做的功能 | 原因 |
|------------|------|
| **播客播放器** | 会变成泛播客 App，稀释"学习"定位 |
| **音频上传 + STT** | MVP 阶段成本/开发周期不可控，用 YouTube/RSS 字幕替代 |
| **社交/分享/评论** | 分散定位，不是个人学习闭环所需 |
| **复杂知识图谱** | 很酷但重，不影响第一版学习闭环 |
| **手动笔记编辑器** | 会和 Notion 竞争，我们做的是 AI 自动生成 |
| **多人协作** | 不是个人学习 MVP 核心 |
| **复杂标签系统** | 增加整理负担，MVP 用固定分类 |
| **多 AI 模式切换** | 用户会选择困难，只做一种最优流程 |
| **微信/邮件推送** | 复习提醒 MVP 只在网站内展示 |
| **RSS 订阅整个播客** | 我们是"单集深度学习"，不是"泛听工具" |

## Primary User Flow 主流程

一次完整学习闭环（10-15 分钟）：

```
1. 打开 K0 → Learn
2. 粘贴 YouTube 链接
3. 系统抓取字幕（10-30 秒 loading）
4. 5 个学习目标按钮 → 用户点 "深度学习"
5. 系统生成学习快照（一句话、3 观点、价值分、成本）
6. 用户看快照决定：跳过 / 3 分钟速学 / 15 分钟精学
7. 选精学 → 进入学习路径 6 步
8. 每步展示核心内容 + 打勾
9. 步骤 3-4 之间生成 5 张知识卡片
10. 用户逐张浏览，收藏想留的
11. 学习路径结束 → 3 道测验题
12. 完成测验 → 生成行动清单（今天/本周/长期）
13. 系统自动加入复习队列（明天/三天/一周）
14. 用户回到首页
```

第二次访问：
```
1. 打开 K0 → Review 显示 "今天有 5 张卡片需要复习"
2. 用户点开 → 逐张回忆 → 记得/不记得
3. 记得 → 下次间隔加长；不记得 → 明天再来
```
