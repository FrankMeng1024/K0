## §chosen-style — 确认选定风格

**Style F — Cutout Illustrated**
文件：`style-demos/k0-demo-f-cutout-illustrated.html`（5 页 showcase：Home / Learn / Episode / Review / Library）

**用户确认于**：2026-07-04（`/project --auto` CP1 通过）

**用户附加要求**：
> "真正开发的时候可以更细致 抽象一些"

**解读（Arch）**：
- 保留 F 的核心视觉基因：撕纸剪贴 / 胖字母 / 有机色块 / 牛皮纸背景 / 织布条 / 白色椭圆气泡
- 但在真实产品实现中：
  1. **细致**：SVG 插画增加更多描述层次（表情、动作、光影暗示），不再是"色块+简线"
  2. **抽象**：物件更符号化、几何化，减少写实还原（例如"耳机"不做写实 headphone，做几何抽象耳机符号）
  3. **原型灵活性**：demo F 是概念定型，最终实现可根据实际内容密度调整插画数量和位置

**核心视觉锁定**：
| 元素 | 值 | 用途 |
|------|-----|------|
| 主色板 | `#C80306` 砖红 · `#284EA9` 宝蓝 · `#F8D34A` 蛋黄 · `#652300` 深棕 · `#C14F94` 玫红 · `#6B6A4E` 橄榄 | 分类标签、卡片背景、CTA |
| 纸质基底 | `#E8D9B8` 主 · `#DDCEA9` 深 · `#F5EBD3` 米黄内层 | 页面背景 |
| Ink | `#1A1613` 主文字 · `#4A3F35` 次文字 | 文本 |
| 主字体 | Bagel Fat One | 大标题 (h1/h2 hero) |
| 辅助字体 | Rubik Bubbles | 泡泡字母（可选替换） |
| UI 字体 | Sniglet | 按钮、chip、UI 组件 |
| 正文字体 | Fraunces（Italic） | 副标题、lead、卡片描述 |
| 撕纸滤镜 | `feTurbulence baseFrequency=0.045 numOctaves=3 seed=4 scale=9` (strong) / `0.06/2/7/5` (mid) / `0.08/2/2/2.5` (soft) | 所有色块边缘 |
| 织布条 | 6 色 warp/weft 双向 repeating-linear-gradient | 分隔符 |
| 白色椭圆气泡标签 | `border-radius: 100px` + 圆点前缀 + 手写体 italic | 提示、状态 |

**禁用元素**（保持风格纯度，不允许在生产代码中出现）：
- `box-shadow` 有偏移量的（neo-brutalism 特征）
- 粗描边（> 2px 的 border）
- 渐变（linear-gradient/radial-gradient）——纯色平涂
- 圆角 border-radius > 16px 且带阴影（现代 SaaS 特征）
- 不带纹理的纯白/纯灰背景

## Product Soul（产品灵魂）

**情感核心**: 专注 / 可完成 / 值得信赖

用户来 K0 的时候是"我今天要真正学到点东西"，不是"我随便看看"。产品要传递：
- 你在学习，不在消费
- 每一步都能被完成，不会中途卡住
- 系统认真处理你的时间与注意力

**视觉隐喻**:
- 学习卡片是"可以完成的任务"，不是"堆积的信息"
- 学习路径像"一节课", 6 步是清晰的节奏
- 复习队列是"陪伴", 不是"任务清单式的压迫"

**交互故事**:
1. 用户粘贴链接 → 系统立刻反馈"正在为你抓取"
2. 学习包生成过程可见（进度、当前步骤）
3. 每完成一步，有清晰的完成动画（不炫技，只是"打勾"）
4. Review 打开时先安抚："今天只有 5 张，很快"

**字号阶梯**:

| `text-h1` | 24px / 1.3 | 页面标题 |

| `text-small` | 13px / 1.5 | 元信息 |
| `text-xs` | 12px / 1.4 | 时间戳、标签 |

### 间距

Tailwind 默认阶梯，重点使用: `2 / 3 / 4 / 6 / 8 / 12 / 16 / 20 / 24`（对应 8/12/16/24/32/48/64/80/96 px）。

**布局网格**:
- 移动 375px：外边距 16px，内容宽 343px
- 桌面 1280px：外边距 96px，内容最大宽度 1088px，主布局 max-width 720px（阅读友好）

### 圆角

- 卡片: `12px`
- 按钮: `8px`
- 输入框: `8px`
- 标签芯片: `full`（药丸）
- 图标按钮: `full`

### 阴影

统一走 3 阶：`--shadow-sm / md / lg`。禁止发明新阴影。

### 图标

**规则**（同 FRONTEND_STANDARDS 图标系统）:
- 全站只用 `lucide-react`
- 尺寸: 16 / 20 / 24 / 32 四档
- 线宽: `1.75px`（默认 lucide 是 2px，稍细）
- 图标 + 文字组合时，图标始终左侧、垂直居中

## 通用组件规范

### 按钮
- **Primary**: 品牌色实心，14px 文字，高 44/48 px，圆角 8px
- **Secondary**: 品牌色边框 + 白底
- **Ghost**: 无边框，仅文字 + hover 浅底
- **Danger**: 红橙色（删除卡片）

### 输入框
- 高 44px，圆角 8px，边框 `--border-default`
- Focus: 边框变品牌色 + 2px 内阴影
- Placeholder: `--text-tertiary`

### 卡片
- 白底，圆角 12px，`--shadow-sm`
- 内边距 16-20 px
- 移动占满宽度

### 加载状态
- 骨架屏（skeleton）而不是 spinner（更符合"专注"氛围）
- 进度型任务用带步骤的 checklist（如导入进度）

### 反馈
- 成功: 顶部 toast 3 秒自动消失，`--success`
- 错误: 内联在相关元素旁，`--danger`，不用 alert()
- 空状态: 中央插图 + 一句引导 + 主要动作按钮

## 动效原则

**尊重时间，不炫技**。所有动画:
- 时长 ≤ 300ms
- Easing: `cubic-bezier(0.4, 0, 0.2, 1)`（标准 Material easing）
- 只在状态变化时用（打勾、翻卡、进入下一步）
- 禁止：轮播、抖动、粒子、渐变色 badge

**允许的动画**:
1. 打勾动画：SVG stroke-dashoffset 200ms
2. 卡片翻转：3D rotate Y，250ms
3. 页面切换：淡入淡出 200ms
4. Skeleton 呼吸：1.5s 循环

## 差异化视觉记忆点

三个刻意保留的"K0 味道"：
1. **serif "这集一句话"**：所有播客快照的核心一句用衬线字体，像书里的引言
2. **6 步竖向进度条**：不是数字条，而是"课程进度"的感觉
3. **卡片左侧类型色条**（4px 宽）：卡片主体保持干净白，仅左侧用颜色标类型
