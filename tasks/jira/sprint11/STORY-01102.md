# STORY-01102: 快照页新建

**Epic**: E-002 · **Sprint**: 11 · **Points**: 3 · **Owner**: Frontend · **Status**: Todo

## Description
新建 `app/snapshot/[packId].tsx`，展示 Step 1 GLM 生成的快照，让用户 10 秒判断+决策。

## Expected Result
用户从 import 屏完成后跳到快照页，从上到下看到：原文(收起)→ 元信息 → 一句话 → 价值分 → 学习成本 → audience → 观点list → skippable → 3 决策按钮。

## Acceptance Criteria
- [ ] 新建 `app/snapshot/[packId].tsx` 路由
- [ ] 顶部使用 ScreenHeader 组件（返回 + "快照" + "10 秒判断"副标题 + 分割线）
- [ ] 从上到下 8 个区块：
  - [ ] 原文（顶端，默认收起，展开后按 15min 章节分节，右下"折叠"）
  - [ ] 元信息卡（封面 + 播客名 + 集标题 + 时长 + 语言）
  - [ ] 一句话总结（≤25 字，大字号 Bagel Fat One）
  - [ ] 价值分（3 条撕纸风进度条：信息密度/新观点/可行动性）
  - [ ] 学习成本（"预估 X 分钟能学完"）
  - [ ] 适合谁学（audience list，撕纸 chip）
  - [ ] 值得听的段（不限 3 个动态渲染，每条 观点+第X分钟+一句话说明+可折叠原文引用段）
  - [ ] 可跳过段（次要区块，色淡，时间戳 chip 列表）
- [ ] 底部固定 3 决策按钮：
  - [ ] "跳过"（brick 红中性）→ POST /api/packs/:id/generate {mode:'skip'} → 回首页
  - [ ] "速学"（yolk 黄）→ POST /api/packs/:id/generate {mode:'quick'} → loading → 跳 episode/[id]?mode=quick
  - [ ] "精学"（sapphire 蓝）→ POST /api/packs/:id/generate {mode:'deep'} → loading → 跳 episode/[id]?mode=deep
- [ ] Playwright QA: 390x844 视口 0 console errors，能看到完整快照信息

## Dependencies
- 依赖 STORY-01101 Backend 拆两步（消费 GET /api/packs/:id 的 snapshot 字段 + POST /generate）
- 依赖 STORY-01106 ScreenHeader 组件
