# STORY-01101: Backend 拆两步 GLM API

**Epic**: E-002/E-003 · **Sprint**: 11 · **Points**: 3 · **Owner**: Backend · **Status**: Todo

## Description
把当前"一次 GLM 全跑"的 pipeline 拆为 Step 1 (快照) + Step 2 (学习包)，只在用户明确选"速学/精学"后才调 Step 2。

## Expected Result
- `POST /api/episodes/import-url` pipeline 只跑到 Step 1（snapshot 生成），返回 job status = ready 时快照已存
- 新 endpoint `POST /api/packs/:id/generate` (body: `{ mode: 'quick' | 'deep' }`) 触发 Step 2 学习包生成
- 学习包 pack_json 结构升级（含 snapshot + steps + concepts + cards[8字段] + actions）
- 删除内部 quiz 生成（generateLearningPack 中的 quizQuestions 逻辑）

## Acceptance Criteria
- [ ] `backend/src/services/packGenerator.js` 拆为 `generateSnapshot(segments, language)` + `generateLearningPack(snapshot, mode)`
- [ ] `backend/src/routes/importUrl.js` 的 runPipeline 到"生成快照存 pack_json.snapshot"就 completeJob，不再调 Step 2
- [ ] 新增 `backend/src/routes/packs.js` 内的 `POST /api/packs/:packId/generate` 路由（body: {mode}）
  - 校验 mode 只能 quick 或 deep
  - 从 DB 读 pack_json.snapshot（不存在则 404）
  - 调 generateLearningPack(snapshot, mode) → 合并 pack_json 存回
  - 返回完整 pack_json
- [ ] `learning_packs.pack_json` schema 支持 snapshot / steps / concepts / cards[8字段] / actions
- [ ] 删除 pack_json.quizQuestions（不再生成）
- [ ] 从 buildMockPack (no-DB fallback) 也更新到新结构
- [ ] SPIKE-010 验证过：连续 3 轮 0 次 429

## Dependencies
- 依赖 STORY-01107 Prompt v2（Step 1/Step 2 独立 prompt）

## Notes
- OTA-safe: backend 改动，需要 scp + systemctl restart，无 OTA
- 现有 429 fallback/cooldown 逻辑（Sprint 10 v16）保留作为最后防线
