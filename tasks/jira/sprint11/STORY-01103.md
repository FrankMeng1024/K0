# STORY-01103: 学习包页重构（quick/deep 共用）

**Epic**: E-003/E-004 · **Sprint**: 11 · **Points**: 3 · **Owner**: Frontend · **Status**: Todo

## Description
重构 `app/episode/[id].tsx`，从"tab 切换 (原文/学习内容)"改为"上下堆叠 (原文顶端 + 6步 + 概念 + 卡片 + 行动)"。quick/deep 共用页面，通过 URL query `mode` 参数控制内容深度。

## Expected Result
用户点速学/精学 → 学习包页从上到下：原文(收起 + 精简/全文toggle) → 6 步引导 → 概念解释器 → 卡片 → 行动清单。所有卡片使用新 8 字段结构。

## Acceptance Criteria
- [ ] 顶部使用 ScreenHeader 组件（返回 + mode=quick 显示"速学"/ mode=deep 显示"精学" + 副标题=播客集标题 + 分割线）
- [ ] 从上到下 5 大区块（垂直堆叠，不用 tab）：
  - [ ] **区块 1 原文**：默认收起。展开时顶部 toggle "精简 / 全文"（默认精简 = 去 skippable 段）。按 15min 章节分节。右下"折叠"按钮
  - [ ] **区块 2 6 步引导路径**（M4-A）：6 张撕纸风小卡垂直堆叠，每步 title + content + checkbox。底部 "N/6 已完成" 进度条。勾选后 PATCH /api/steps/:id 持久化
  - [ ] **区块 3 概念解释器**（M4-B）：可点击 chip 列表，点开弹三层解释（plain + context+timestamp + related）
  - [ ] **区块 4 知识卡片**（M4-C 8 字段）：quick=3-5 张, deep=3-18 张。每张：title / type 标签 / core / context / usage / challenge / source（含 quote+timestamp）/ myApplication（可编辑）。右上角 ★收藏 / 🗑删除
  - [ ] **区块 5 行动清单**（M4-D）：3 条（今天/本周/长期），每条前 checkbox，勾选 POST /api/review/actions/commit
- [ ] quick 模式底部显示"升级到精学"按钮 → POST /api/packs/:id/generate {mode:'deep'} → loading → 跳 mode=deep
- [ ] 删除现有的 QuizPanel 组件（走 CR-003）
- [ ] Playwright QA: 390x844，quick + deep 双模式测，0 console errors

- 依赖 STORY-01106 ScreenHeader
