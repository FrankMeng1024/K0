# STORY-00031: 学习包生成 Backend — generate endpoint + job system + GLM prompt

**Epic**: E-003 Learning Pack Generator
**Sprint**: Sprint 3
**Sprint Goal**: 用户可选学习目标并看到 GLM 生成的快照和学习路径
**Points**: 5
**Owner**: Backend
**Status**: Done

## Description
As a user, I want to trigger generation of a learning pack for an episode with a specific goal, so the system can produce a complete snapshot + learning path + cards.

## Expected Result
`POST /api/episodes/:id/generate` accepts a goal, calls GLM to generate a full learning pack (snapshot + 6 steps + 3-5 cards + actions), stores the result in memory (no-DB mode), and returns a jobId. Polling `GET /api/jobs/:jobId` returns status=ready with packId once generation completes. `GET /api/packs/:packId` returns the full PackObject.

## Acceptance Criteria
- [x] `POST /api/episodes/:id/generate` accepts `{ "goal": "quick_understand|deep_learn|find_actions|critical_thinking|for_work" }` and returns `{ "jobId": "<uuid>", "status": "processing" }`
- [x] Invalid goal value → 400 VALIDATION_ERROR with message
- [x] Non-integer episode ID → 400 VALIDATION_ERROR
- [x] `GET /api/jobs/:jobId` returns `{ "status": "processing|ready|failed", "progress": 0-100, "packId": N }` — packId present when status=ready
- [x] Unknown jobId → 404 NOT_FOUND
- [x] `GET /api/packs/:id` returns PackObject with all required fields: `id, episodeId, goal, language, snapshot{ oneSentence, corePoints[3], audience, valueScore{density,novelty,actionability}, estimatedCostMinutes }, steps[6], cards[3-5], actions{ today, thisWeek, longTerm }`
- [x] GLM prompt generates content in the episode's language (zh or en) based on `episode.language`
- [x] With invalid/placeholder GLM API key → 502 GLM_API_ERROR (same as STORY-00020 pattern)
- [x] In no-DB mode (db=null): generate endpoint uses mock episode data, generates real GLM call if key is valid; if GLM fails, returns 502; job status transitions from processing → failed correctly
- [x] Unit test: validates PackObject shape from GLM response parsing
- [x] Unit test: validates goal enum rejection

## Dependencies
- Depends on: STORY-00033 (DB schema), STORY-00020 (GLM service pattern)
- Blocks: STORY-00032

## Notes
**GLM Prompt Strategy**: Single prompt approach — pass episode transcript/text + goal → GLM returns full JSON with all 5 sections. Prompt files: `backend/prompts/generate-pack.zh.md` + `generate-pack.en.md`.

**No jobs queue**: Backend synchronously awaits GLM response. Job entry created before GLM call (status=processing), updated after (status=ready/failed). In-memory job store (Map) sufficient for MVP single-user.

**Job TTL**: Jobs expire after 1 hour (cleanup on read).

**Rate limit**: inherit existing `POST /api/episodes/:id/snapshot` limit (5/hour/user_id).
