# STORY-00021: 快照 UI 卡片 + 单集详情页骨架

**Epic**: E-002 Learning Snapshot
**Sprint**: Sprint 3
**Sprint Goal**: 用户可选学习目标并看到 GLM 生成的快照和学习路径
**Points**: 5
**Owner**: Frontend
**Status**: Todo

## Description
As a user, after selecting my learning goal, I want to see a visual episode detail screen with a snapshot card showing the one-sentence summary, core points, and value scores.

## Expected Result
Episode detail screen (`/episode/[id]`) loads with: top status bar (title + status chip), and a snapshot section showing the GLM-generated snapshot. While snapshot is loading (job in progress), show a skeleton/loading state.

## Acceptance Criteria
- [ ] `app/episode/[id].tsx` exists and is reachable via Expo Router from GoalSelect
- [ ] Episode screen shows: back button (canGoBack pattern), episode title, channel, import status chip
- [ ] Snapshot section renders when `GET /api/packs/:id` resolves: one-sentence quote (serif font, large), 3 core points list, value score bars (density/novelty/actionability each 1-10), estimated cost minutes
- [ ] While loading (job processing): skeleton placeholders shown for snapshot content (not empty, not spinner — per UI_SPEC loading standards)
- [ ] If generation fails (status=failed): show error state "学习包生成失败，请重试" with a retry button
- [ ] 3 action buttons below snapshot: "3 分钟速学" (no-op placeholder) · "完整学习" (scrolls to steps section) · "跳过这集" (navigate back to home)
- [ ] "跳过这集" uses `router.replace('/')` (history-safe)
- [ ] No console errors on 375px and 414px viewports
- [ ] Screenshot shows styled snapshot card matching UI_SPEC §单集页面 first screen design

## UX ACs
- [ ] The one-sentence summary is visually prominent (largest text on screen, serif font)
- [ ] Value score bars are immediately readable — no legend needed
- [ ] Loading state is calm (skeleton, not spinner) — communicates "working on it" not "broken"
- [ ] All 3 action buttons visible without scrolling on 375px viewport

## Dependencies
- Depends on: STORY-00030 (GoalSelect provides episodeId + goal), STORY-00031 (provides pack data)
- Blocks: STORY-00032

## Notes
Episode screen polls `GET /api/jobs/:jobId` every 2 seconds until status ≠ processing (max 90s timeout). On timeout: show error state.
Job polling is initiated after navigating to the Episode screen with `jobId` as a route param (GoalSelect passes it after triggering generate).
Actually: GoalSelect calls `POST /api/episodes/:id/generate`, gets jobId, then navigates to `/episode/[id]?jobId=xxx`. Episode screen receives jobId and polls.
