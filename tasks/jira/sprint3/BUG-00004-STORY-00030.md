# BUG-00004-STORY-00030: GoalSelect 从 Episode 返回后所有目标按钮被禁用

**Priority**: Medium
**Sprint**: Sprint 3
**Opened by**: QA (Stage 1 smoke test, post-BUG-00003 flow)
**Status**: Verified

## Description
用户从 Episode 详情页点击返回后回到 GoalSelect 屏，5 个学习目标按钮全部 disabled 且之前选中的按钮显示 progressbar，无法重新选择目标。

## Steps to Reproduce
1. Home → Learn → 粘贴文本 → 开始 → 下一步 → 选择任一目标
2. 进入 Episode 详情页，等待 pack 生成完成
3. 点击左上「← 返回」

## Expected
GoalSelect 显示 5 个可点击按钮，用户可再选一个新目标。

## Actual
5 个按钮全部 `disabled`，第一个（原选中的）显示 progressbar 转圈，用户被卡住。

## Root Cause
`app/goal-select.tsx` 的 `onSelectGoal` 在成功路径 `router.push('/episode/...')` 后**没有清空** `loading` / `loadingGoal` state（第 79-87 行）。React Native 屏幕栈保留组件实例，goBack 时 state 保留。用户从 Episode 返回时 GoalSelect 组件重新 focus 但 state 仍是 `loading=true`。

## Fix
`app/goal-select.tsx`：
1. 从 `expo-router` 引入 `useFocusEffect`
2. 在组件内添加：
```ts
useFocusEffect(
  useCallback(() => {
    setLoading(false);
    setLoadingGoal(null);
    setError(null);
  }, [])
);
```
这样每次 GoalSelect 重新 focus（首进 + goBack）都重置本地 state。
