# STORY-00032: 单集详情页 — 6 步学习路径 + 知识卡片列表

**Epic**: E-004 Learning Player
**Sprint**: Sprint 3
**Sprint Goal**: 用户可选学习目标并看到 GLM 生成的快照和学习路径
**Points**: 5
**Owner**: Frontend + Backend
**Status**: Todo

## Description
As a user, I want to scroll through the 6-step learning path and knowledge cards on the episode detail page, and mark steps as completed, so I can experience a full structured learning session.

## Expected Result
Below the snapshot section, the episode detail screen shows: 6 numbered learning steps (expandable, with content), a knowledge cards section (3-5 cards with type color-coding), and an actions section (today/this-week/long-term). Steps can be toggled complete via PATCH /api/steps/:id.

## Acceptance Criteria
- [ ] Learning path section shows 6 steps with: step number, title, completed checkbox/indicator
- [ ] Tapping a step expands to show its content (plain text, no markdown rendering required in Sprint 3)
- [ ] Tapping the completion indicator on a step calls `PATCH /api/steps/:id` with `{ completed: true }` and updates visual state
- [ ] In no-DB mode: PATCH /api/steps/:id returns updated step object with `completed: true` (in-memory state)
- [ ] `PATCH /api/steps/:id` non-integer ID → 400 VALIDATION_ERROR
- [ ] Knowledge cards section shows 3-5 cards with: type color bar (left 4px per UI_SPEC), title, explanation, source timestamp (seconds → mm:ss formatted), starred indicator
- [ ] Actions section shows 3 action items: today / this week / long term
- [ ] Page scrolls smoothly through all sections
- [ ] No console errors during scroll and step completion interactions on 375px viewport

## UX ACs
- [ ] Completed steps are visually distinct from incomplete (checkmark + muted color)
- [ ] Card type color bars are visible and match UI_SPEC color coding (opinion=blue, method=green, case=orange, reflection=purple, action=red)
- [ ] Each card shows enough information to understand the insight without tapping

## Backend ACs
- [ ] `PATCH /api/steps/:id` endpoint exists, returns updated LearningStepObject
- [ ] `PATCH /api/steps/:id` in no-DB mode: updates in-memory state on the pack mock data

## Dependencies
- Depends on: STORY-00021 (episode screen exists), STORY-00031 (pack data available)
- Blocks: none (Sprint 4 features depend on this)

## Notes
Timestamp display: `Math.floor(ts / 60)` + ':' + `String(ts % 60).padStart(2, '0')` — no library needed.
No PATCH /api/cards/:id in Sprint 3 (star/delete deferred to Sprint 4).
