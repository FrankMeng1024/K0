# STORY-00033: DB Migrations — Learning Pack Tables + Mock Data

**Epic**: E-003 Learning Pack Generator
**Sprint**: Sprint 3
**Sprint Goal**: 用户可选学习目标并看到 GLM 生成的快照和学习路径
**Points**: 2
**Owner**: Backend
**Status**: Done

## Description
As a developer, I need the database tables for learning_packs, learning_steps, snapshots, and cards to exist, so that Sprint 3 backend stories can save and retrieve learning pack data.

In no-DB mode, backend must return a fixed mock PackObject for any `GET /api/packs/:id` request to unblock frontend development.

## Expected Result
- SQL migration files exist for all 4 new tables
- `backend/migrations/004_learning_packs.sql` creates: `learning_packs`, `snapshots`, `learning_steps`, `cards`, `quizzes`
- no-DB mock: `GET /api/packs/:id` returns a valid PackObject with mock snapshot + 6 steps + 3 cards
- Tests verify migration SQL syntax is valid

## Acceptance Criteria
- [ ] `backend/migrations/004_learning_packs.sql` exists and creates all required tables per DB_SCHEMA.md schema

- [ ] Each table has `user_id`, `created_at`, `updated_at` where specified in schema
- [ ] no-DB mock response for `GET /api/packs/1` returns: `{ id, episodeId, goal, language, snapshot: { oneSentence, corePoints[3], valueScore }, steps: [6 LearningStepObjects], cards: [3 CardObjects] }`
