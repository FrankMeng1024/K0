# STORY-00030: 学习目标选择 UI（5 选 1）

**Epic**: E-003 Learning Pack Generator
**Sprint**: Sprint 3
**Sprint Goal**: 用户可选学习目标并看到 GLM 生成的快照和学习路径
**Points**: 3
**Owner**: Frontend
**Status**: Done

## Description
As a user, I want to choose my learning goal after importing a podcast episode, so that the AI can tailor the learning pack to my specific intent.

## Expected Result
After EpisodeCard is shown and user taps "下一步", a goal selection screen appears with 5 clearly labeled buttons. User taps one, which navigates to the episode detail screen with the selected goal as a parameter.

## Acceptance Criteria
- [x] After EpisodeCard appears, "下一步" button (currently placeholder) becomes functional and navigates to a GoalSelect screen/modal
- [x] GoalSelect screen shows 5 goal buttons in a vertical list: "⚡ 快速了解" / "🎯 深度学习" / "⚙ 找可执行方法" / "🔍 批判性思考" / "📎 为工作/研究使用"
- [x] Each button ≥ 56px height (touch-friendly per UI_SPEC M2 Success Criteria)
- [x] All 5 buttons visible without scrolling on 375px viewport
- [x] Tapping a goal button navigates to Episode screen (`/episode/[episodeId]`) with `goal` as route param
- [x] GoalSelect screen has a back button ("← 返回") that uses `canGoBack() ? back() : replace('/')`
- [x] No console errors during full navigation: Home → Learn → import → 下一步 → GoalSelect → select goal → Episode screen
- [x] Goal buttons styled per UI_SPEC: primary-style or distinct card buttons; wind metaphor colors from chosen style

## UX ACs
- [ ] Goal labels are self-explanatory to a first-time user (no implementation knowledge needed)
- [ ] After tapping a goal, there is immediate visual feedback (button highlight) before navigation starts
- [ ] Navigation completes < 500ms (no spinner needed for local navigation)

## Dependencies
- Depends on: STORY-00010 (EpisodeCard), STORY-00021 (Episode screen target)
- Blocks: none

## Notes
GoalSelect can be a new screen (`app/goal-select.tsx`) or a modal sheet. Arch recommendation: new screen for simplicity with Expo Router.
Episode route: `app/episode/[id].tsx` — created in STORY-00021.
