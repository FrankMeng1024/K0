# STORY-00002: Backend 骨架（Express + mysql2）+ auth middleware + user_id 预留

**Epic**: E-007 Foundation（CR-001 后重写）
**Sprint**: Sprint 1
**Points**: 3
**Owner**: Backend
**Status**: Todo

## Description
As Backend, I want to 在 `backend/` 子目录建 Express 5 + mysql2 + JWT middleware 骨架（参考但不复制 Cairn 架构），so that Sprint 2+ 业务 API 有基础。

## Expected Result
- `backend/package.json`（Express 5, mysql2, jsonwebtoken, zod, helmet, cors, express-rate-limit, pino, node-cron, dotenv）
- `backend/src/index.js` 启动入口（端口 3002 —— 不与 Cairn 3001 冲突）
- `backend/src/config/db.js` mysql2 连接池
- `backend/src/middleware/auth.js`（JWT 中间件，默认放行 user_id=1 if `AUTH_ENABLED=false`）
- `backend/src/routes/health.js` `/health` endpoint
- `backend/src/routes/whoami.js` `/api/whoami` 返回当前 user_id（测试用）
- `backend/migrations/001_init.sql` 建 users 表 + 默认 user_id=1
- `backend/scripts/migrate.js` 执行 migration
- `backend/Dockerfile` 沿用 Cairn 模式但更精简
- `.env.example` 完整（DB_HOST/PORT/USER/PASSWORD/NAME, JWT_SECRET, GLM_API_KEY, CORS_ORIGINS）
- 单元测试：mock header 有 valid JWT → 拿 user_id；无 header → 拿 1

## Acceptance Criteria
- [ ] `cd backend && npm run dev` 启动成功
- [ ] `curl http://localhost:3002/health` 返回 `{"status":"ok","ts":...}` 200 < 100ms
- [ ] `curl http://localhost:3002/api/whoami` 无 header 返回 `{"user_id":1}`
- [ ] `curl -H "Authorization: Bearer <valid>" http://localhost:3002/api/whoami` 返回对应 user_id
- [ ] MySQL migration 001 可 apply 也可 rollback
- [ ] Docker build 通过（本地 `docker build` 成功）
- [ ] helmet + cors + rate-limit 中间件都激活
