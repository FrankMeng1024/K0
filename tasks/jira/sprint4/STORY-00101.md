# STORY-00101: Home 底部固定 primary CTA "粘贴链接"

**Epic**: E-005 UX Polish
**Sprint**: Sprint 4
**Points**: 3
**Owner**: Frontend
**Status**: Done

## Description
As an iPhone user opening the app, I want to paste a podcast link directly from Home without first tapping "Learn", so the primary action lands in my thumb-reachable zone (bottom 1/3 of screen).

## Expected Result
Home 页底部（tab bar 之上）固定一个粘贴输入框卡片，占屏幕宽度全宽、离底部安全区约 spacing.lg。用户在 Home 就能粘贴链接直接进入 Learn 流程，不必先点导航卡片。原有 3 个入口卡片保留，作为次要入口上移。

## Acceptance Criteria
- [ ] Home 底部（safe area 之上 16pt）新增 `<HomePasteBar />` — 内含"粘贴播客链接..."输入框 + 右侧圆角按钮"→"
- [ ] 输入框获得焦点后底部弹出键盘，粘贴内容后按钮变主色（brick）可点
- [ ] 点击按钮 → `router.push('/learn?text=' + encodeURIComponent(text))`
- [ ] Learn 页读取 `useLocalSearchParams().text`，如存在则预填输入框（覆盖之前 clipboard 逻辑）
- [ ] Home 3 张入口卡片保留（Learn/Review/Library），滚动可见，非阻挡 PasteBar
- [ ] iPhone SE 视口下 PasteBar 不覆盖 Learn 卡片主要内容
- [ ] 三档视口截图均验证拇指区可达

## Dependencies
None
