# STORY-00013: 语言自动识别 + Episode language 字段回填

**Epic**: E-001 Import & Transcribe
**Sprint**: Sprint 2
**Sprint Goal**: 用户可以在 Learn 屏幕粘贴一条 Apple Podcasts 链接或任意文本，看到系统在 5 秒内抓取到播客元数据，并把它保存为一集"待学习"记录。
**Points**: 3
**Owner**: Backend
**Status**: Todo

## Description
As Backend, I want to 在 `POST /api/episodes/import` 处理阶段自动识别 language（`en`|`zh`|`unknown`）并写入 episode 记录。so that 后续 GLM 快照生成（STORY-00020）能选对提示词语种。

## Expected Result
- `backend/src/services/langDetect.js`：
  - `detectLanguage(text)` → `'en' | 'zh' | 'unknown'`
  - 规则实现（无 heavy ML dep）：
    - CJK Unicode (U+4E00–U+9FFF) 字符占比 > 30% → `'zh'`
    - ASCII letters 占比 > 60% → `'en'`
    - 否则 → `'unknown'`
    - 输入 < 20 字 → 直接 `'unknown'`
- `backend/src/routes/episodes.js` 集成：
  - `source='text'` 分支：调 `detectLanguage(body.text)` → 写 `episodes.language` + `transcripts.language`
  - `source='apple'` 分支：从 episode description（iTunes API 返回）调 `detectLanguage`；description 为空则用 podcast feedUrl 拉 RSS 首 500 字检测
- 单元测试 `backend/test/langDetect.test.js`：
  - 纯中文文本 → `'zh'`
  - 纯英文文本 → `'en'`
  - 混合 ≥ 30% CJK → `'zh'`
  - 短文本 < 20 字 → `'unknown'`
  - 全数字/符号 → `'unknown'`

## Acceptance Criteria

### 功能
- [ ] `POST /api/episodes/import { source: 'text', text: '<≥200字中文>' }` → response `episode.language='zh'`
- [ ] `POST /api/episodes/import { source: 'text', text: '<≥200字英文>' }` → response `episode.language='en'`
- [ ] Apple URL import + description 有中文 → `language='zh'`
- [ ] 短文本 < 20 字 → `language='unknown'`（不报错）
- [ ] STORY-00010 前端 language chip 显示 zh/en/未识别 三态之一

### 测试
- [ ] `backend/test/langDetect.test.js` 5+ 断言全绿

### 无功能
- [ ] 无新 npm 依赖（纯 regex/Unicode 判断）
- [ ] 单次 detectLanguage 调用 < 50ms（10000 字文本）
