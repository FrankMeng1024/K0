# STORY-00020: GLM 生成学习快照（一句话 + 3 观点 + 价值分）

**Epic**: E-002 Learning Snapshot
**Sprint**: Sprint 2
**Sprint Goal**: 用户可以在 Learn 屏幕粘贴一条 Apple Podcasts 链接或任意文本，看到系统在 5 秒内抓取到播客元数据，并把它保存为一集"待学习"记录。
**Points**: 5
**Owner**: Backend
**Status**: Todo

## Description
As User, I want to 对一集已导入的播客（text 分支或 apple 分支带 transcript）调用一次 API，20 秒内拿回"一句话总结 + 3 个核心观点 + 价值分（密度/新鲜/可行）"，用于后续 UI 卡片。so that Sprint 3 的 Learning Pack 生成有起点。

## Expected Result
- `backend/src/services/glm.js`：
  - `generateSnapshot({ text, language })` → SnapshotObject（契约见 API_SPEC.md §SnapshotObject）
  - GLM-4-plus JSON schema 模式（SPIKE-003 已验证）
  - 超时 25 秒；失败降级 `{ error: 'GLM_TIMEOUT' | 'GLM_MALFORMED_JSON' }`
  - Prompt 按 language 分中英两个模板：`backend/prompts/snapshot.zh.md` + `backend/prompts/snapshot.en.md`
- `backend/src/routes/snapshots.js`：
  - `POST /api/episodes/:id/snapshot` handler（auth 中间件保护）
  - 从 `transcripts` 表拉 text；无 transcript → 400 `NO_TRANSCRIPT`
  - 调 `generateSnapshot`
  - 写 `snapshots` 表（migration 003）：id, episode_id, snapshot_json JSON, language, created_at
  - 幂等：同一 episode 二次调用返回已有 snapshot（除非 `?regenerate=true`）
  - 返回 SnapshotObject
- `backend/migrations/003_snapshots.sql`：建 `snapshots` 表
- Rate limit：`POST /api/episodes/:id/snapshot` 单用户 5/小时
- 单元测试：
  - service 层用 mock GLM response（SPIKE-003 fixture）
  - Route 契约测试
  - 401 negative path
  - 无 transcript → 400 `NO_TRANSCRIPT`
  - GLM timeout mock → 502 `GLM_TIMEOUT`

## Acceptance Criteria

### 功能
- [ ] `POST /api/episodes/:id/snapshot`（episode 有 transcript ≥ 500 字）→ ≤ 20 秒 200 + SnapshotObject
- [ ] SnapshotObject：`oneSentence` (≤25 字), `corePoints` (3 项), `valueScore.density/novelty/actionability` (1-10 int)
- [ ] 中文 episode → oneSentence 是中文
- [ ] 英文 episode → oneSentence 是英文
- [ ] 二次调用（无 regenerate）→ 返回缓存，response < 200ms
- [ ] `?regenerate=true` → 重新调 GLM，覆盖 snapshots 表

### 错误
- [ ] 无 transcript → 400 `NO_TRANSCRIPT`（契约 envelope）
- [ ] GLM 超时 → 502 `GLM_TIMEOUT`（契约 envelope）
- [ ] GLM 返回非法 JSON → 502 `GLM_MALFORMED_JSON`（契约 envelope）

### 契约
- [ ] SnapshotObject 与 `docs/API_SPEC.md` §SnapshotObject 完全一致
- [ ] Rate limit 5/hour keyed by user_id → 429 契约 envelope

### 测试
- [ ] 单元测试全绿
- [ ] （无前端 Playwright 测试本 Sprint —— STORY-00021 Sprint 3 才建 UI）

### 性能
- [ ] p95 < 20 秒（SPIKE-003 median 8-15 秒）
