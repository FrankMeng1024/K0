# STORY-00090: Sprint 1 Carry-Forward — AUTH 生产守卫 + 错误 envelope + API_SPEC 文档更新

**Epic**: E-007 Foundation
**Sprint**: Sprint 2
**Sprint Goal**: 用户可以在 Learn 屏幕粘贴一条 Apple Podcasts 链接或任意文本，看到系统在 5 秒内抓取到播客元数据，并把它保存为一集"待学习"记录。
**Points**: 3
**Owner**: Backend + Arch
**Status**: Done

## Description
As Arch, I want to 关闭 Sprint 1 review 的 3 个 medium 问题：
1. `AUTH_ENABLED=false` 无生产守卫 —— 添加 `NODE_ENV==='production'` 强制启用 + 启动 loud warning
2. 错误 envelope `{ error: { code, message, details } }` 未统一 —— 修复 `backend/src/middleware/auth.js` 现在返回的裸字符串 error，改成契约格式
3. `API_SPEC.md` 未文档化 rate-limit 实际阈值 + `/health` 的 `ok|degraded` + `/api/whoami` 的 `source` 字段

so that Sprint 2 业务接口在一个契约稳定的地基上开发。

## Expected Result
- `backend/src/middleware/auth.js` 三处 401 返回都改成 `{ error: { code: 'MISSING_AUTH' | 'INVALID_AUTH_SCHEME' | 'INVALID_TOKEN', message, details } }`
- `backend/src/index.js` 启动时：若 `NODE_ENV==='production'` 且 `AUTH_ENABLED !== 'true'`，覆盖为 `true` 并 `console.warn`（醒目 `[SECURITY]` 前缀）
- 单元测试：新增 `backend/test/auth.negative.test.js` 覆盖 401 每个 code path + 契约格式断言
- `backend/src/lib/errors.js`：定义错误 code 枚举 + `throwApiError(code, message, details?)` helper（复用于所有 route）
- `docs/API_SPEC.md` 更新：
  - `/health` response `{ status: 'ok'|'degraded', db: { ok, error?, latency_ms? } }` 契约固化
  - `/api/whoami` response 加 optional `source: 'jwt'|'dev_default'`
  - §Rate Limits 章节把"Post-MVP 不做"改为"MVP 已启用"，写具体阈值（引用 backend 代码里 express-rate-limit 配置）
- `docs/TECH_SPEC.md` §rate-limit 处同步更新
