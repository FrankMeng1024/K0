# STORY-00011: Apple Podcasts URL 元数据抓取（Backend）

**Epic**: E-001 Import & Transcribe
**Sprint**: Sprint 2
**Sprint Goal**: 用户可以在 Learn 屏幕粘贴一条 Apple Podcasts 链接或任意文本，看到系统在 5 秒内抓取到播客元数据，并把它保存为一集"待学习"记录。
**Points**: 5
**Owner**: Backend
**Status**: Done

## Description
As Backend, I want to 实现 `POST /api/episodes/import` 处理 Apple Podcasts 单集 URL：调 iTunes Search API 拿 podcast + episode 元数据，写入 `episodes` 表，返回 EpisodeObject。so that Learn 屏幕（STORY-00010）能展示"待学习卡片"。

## Expected Result
- `backend/src/services/appleImport.js`：
  - `parseAppleUrl(url)` → `{ podcastId, episodeId }` 或抛 `INVALID_URL`
  - `fetchAppleMetadata({ podcastId, episodeId })` → iTunes Search API 两次调用（复用 SPIKE-002 已验证的 pattern），返回归一化 `{ title, channel, duration, coverUrl, publishedAt, sourceUrl, sourceId, audioUrl }`
  - 超时 10 秒；失败抛 `SOURCE_UNREACHABLE`
- `backend/src/routes/episodes.js`：
  - `POST /api/episodes/import` handler
  - 请求体 zod 验证 `{ url?: string, source?: 'auto'|'apple'|'youtube'|'spotify'|'text', text?: string }`
  - `source='auto'`（或未提供）时按 URL 识别：`podcasts.apple.com` → apple；`spotify.com` → `SOURCE_NOT_SUPPORTED`；`youtube.com|youtu.be` → `YOUTUBE_MANUAL_ONLY`
  - Apple 分支：调 appleImport service，写 `episodes` 表（`user_id=req.user.id`，`import_status='ready_meta_only'`），返回 `{ episode: EpisodeObject }`
  - Text 分支（source='text'）：本 Story 只建骨架，STORY-00010 后端部分补全
- `backend/migrations/002_add_import_fields.sql`：
  - `MODIFY COLUMN import_status ENUM('transcribing','ready','failed','ready_meta_only')`
  - `MODIFY COLUMN source ENUM('youtube','apple','spotify','text')`
  - `ADD COLUMN audio_url VARCHAR(500) NULL`
- `backend/src/lib/errors.js`：错误 code 枚举 + `throwApiError(code, message, details?)` helper
- Rate limit：`POST /api/episodes/import` 单用户 10/分钟（express-rate-limit keyed by `req.user.id`）
- 单元测试：
  - `parseAppleUrl` 正例（≥ 2 URL from SPIKE-002 fixtures）+ 反例
  - `fetchAppleMetadata` 用 SPIKE-002 fixture mock（不打真外网）
  - route 测试：Apple URL 200 + episode 契约；Spotify URL 400 `SOURCE_NOT_SUPPORTED`；YouTube URL 400 `YOUTUBE_MANUAL_ONLY`；无 URL 400 `INVALID_URL`
  - 401 negative-path（Arch M#2）

## Acceptance Criteria

### 功能
- [ ] `curl -X POST http://localhost:3002/api/episodes/import -H 'Content-Type: application/json' -d '{"url":"<apple_podcasts_url>"}'` 返回 200 + EpisodeObject，`import_status='ready_meta_only'`
- [ ] EpisodeObject 含：id, title, channel, duration, coverUrl, publishedAt, sourceUrl, sourceId, language, importStatus
- [ ] Spotify URL → 400 `{ error: { code: 'SOURCE_NOT_SUPPORTED', message: '目前不支持 Spotify' } }`
- [ ] YouTube URL → 400 `{ error: { code: 'YOUTUBE_MANUAL_ONLY', message: '请把 YouTube 描述或字幕文本粘贴过来' } }`
- [ ] 同一 user 相同 (source, source_id) 二次 import → 返回既有 episode（幂等，依赖 unique key）
- [ ] Migration 002 apply/rollback 通过
- [ ] rate-limit：11 次/分钟内 → 429（契约 envelope）

- [ ] 所有错误响应用 `{ error: { code, message, details? } }` envelope

### 测试
