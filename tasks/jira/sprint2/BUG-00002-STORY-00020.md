# BUG-00002-STORY-00020: GLM API错误(401/4xx)返回500而非502

**Priority**: Medium
**Sprint**: Sprint 2
**Opened by**: QA
**Status**: Verified

## Description
`POST /api/episodes/:id/snapshot` 当 GLM API 返回 4xx 错误（如 401 无效密钥）时，后端返回 500 INTERNAL_ERROR 而非 502。

## Steps to Reproduce
1. 设置无效 GLM_API_KEY（如默认的 placeholder 值）
2. `POST /api/episodes/1/snapshot`
3. 观察响应

## Expected
HTTP 502, `error.code` = 上游错误代码

## Actual
HTTP 500, `error.code = "INTERNAL_ERROR"`

## Root Cause
`glm.js` 对非 2xx 响应设置 `err.glmError = 'GLM_API_ERROR'`，但 `handleGlmError()` 只处理 `GLM_TIMEOUT` 和 `GLM_MALFORMED_JSON`，`GLM_API_ERROR` 未被处理，fallthrough 到通用 500 handler。

## Fix
在 `handleGlmError()` 中添加 `GLM_API_ERROR` case，返回 502。
