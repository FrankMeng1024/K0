# STORY-00010: Learn 屏幕 — 粘贴 URL 或文本 → 结果卡片（前后端联调）

**Epic**: E-001 Import & Transcribe
**Sprint**: Sprint 2
**Sprint Goal**: 用户可以在 Learn 屏幕粘贴一条 Apple Podcasts 链接或任意文本，看到系统在 5 秒内抓取到播客元数据，并把它保存为一集"待学习"记录。
**Points**: 5
**Owner**: Frontend + Backend
**Status**: Todo

## Description
As User, I want to 打开 Learn 屏幕，把一条 Apple Podcasts 链接（或一段文本）粘贴进输入框，点"开始"，看到系统 5 秒内返回一张"待学习卡片"（含 title/channel/duration/cover 或语言/字数统计）；卡片右上角有"删除"按钮，卡片下方显示"下一步：生成学习包（即将上线）"占位。so that 我第一次感受到 K0 的核心动作：粘贴 → 收藏。

## Expected Result

### 前端（app/learn.tsx 从 stub 重写为真页面）
- 顶部保留返回箭头 + BubbleTag（文案"今天可以开始一集"，来自 STORY-00091 M4）
- 主标题 "Learn"（Bagel Fat One, brick color）
- 大输入框（TextInput multiline, max 5000 char），placeholder：`粘贴 Apple Podcasts 链接，或直接粘贴一段播客文本 / 转录`
- 主 CTA 按钮 "开始"（brick color, hero font weight，min 44pt touch target）
- URL 判断（regex `^https?://`）：
  - 是 URL → `POST /api/episodes/import { url }`
  - 否则文本 ≥ 200 字 → `POST /api/episodes/import { source: 'text', text }`
  - 否则 < 200 字 → 前端提示"再多贴一些内容，至少 200 字"（不发请求）
- Loading 状态：CTA 按钮 ActivityIndicator + 文案"正在抓取…"
- 成功：CTA 下方渲染 EpisodeCard（title/channel/duration/cover 缩略图/language chip）
- EpisodeCard 右上角"×"删除按钮（清空本次 session 的卡片，不删 DB 记录）
- EpisodeCard 下方"下一步：生成学习包（即将上线）"灰色占位文案
- 失败：错误 code 映射人类文案：
  - `SOURCE_NOT_SUPPORTED` → "目前不支持这个来源。已支持：Apple Podcasts、直接粘贴文本。"
  - `YOUTUBE_MANUAL_ONLY` → "YouTube 链接目前需要粘贴文本。请复制视频描述或字幕过来。"
  - `INVALID_URL` → "这个链接看起来不太对，检查一下？"
  - `SOURCE_UNREACHABLE` → "网络似乎不稳定，稍后再试。"
  - 其他 → "出了点问题，稍后再试"

### 新组件
- `components/EpisodeCard.tsx`：撕纸风格卡片，accentColor=brick，与 Home Learn 卡片视觉一致

### API client
- `lib/api.ts`：封装 `apiPost('/api/episodes/import', body)` + auth header 注入（MVP dev_default user_id=1）
- 抛 typed error `ApiError { code: string, message: string, details?: unknown }`

### 后端（text 分支补全 STORY-00011 骨架）
- `source='text'` 分支完整实现：
  - zod 验证 `{ source: 'text', text: string }` (min 200, max 100000)
  - `duration = Math.round(text.length / 15)` 秒估算
  - 写 `episodes` 表（source='text', title=`文本 · 首 30 字…`, channel=null, source_url=null, language='unknown'）
  - 写 `transcripts` 表（episode_id, text, language='unknown'）
  - 返回 EpisodeObject with `import_status='ready'`

## Acceptance Criteria

### 功能 — URL 分支
- [ ] Playwright web 版：打开 `/learn` → 输入框粘贴已知 Apple URL → 点开始 → ≤ 5 秒内 EpisodeCard 出现，title 非空
- [ ] EpisodeCard 显示 title / channel / duration（HH:MM 格式）/ cover 缩略图
- [ ] Language chip 显示（"未识别" 或 "en"/"zh" 若 STORY-00013 先完成）

### 功能 — 文本分支
- [ ] 粘贴 ≥ 200 字文本 → ≤ 2 秒内 EpisodeCard 显示"文本 · 首 30 字…"
- [ ] 粘贴 < 200 字 → 不发请求，前端即时提示

### 功能 — 错误路径
- [ ] 粘贴 Spotify URL → 显示"目前不支持这个来源…"
- [ ] 粘贴 YouTube URL → 显示"YouTube 链接目前需要粘贴文本…"

### UX
- [ ] Loading 状态可见（spinner + 文案变化）
- [ ] EpisodeCard 出现时 fade-in 200ms 动画
- [ ] Learn 屏幕不含"Sprint 2 会填这里的功能"字样（M3 自然覆盖）
- [ ] iPhone SE 375×667 一屏可见：header + 输入框 + CTA（EpisodeCard 出现后可 scroll）
- [ ] CTA 按钮 touch target ≥ 44pt

### 契约
- [ ] 前端错误映射覆盖 API_SPEC 定义的所有 error code
- [ ] lib/api.ts 遇到非 2xx 抛 ApiError { code, message }

### 无功能
- [ ] `browser_console_messages(level="error")` 空
- [ ] Playwright E2E test 覆盖：URL 成功 / 文本成功 / Spotify 失败 3 条链路

## Notes
- 用户旅程的第一次"我真的在用 K0"—— 视觉/文案不要偷懒
- 文本 200 字下限是 GLM 生成快照的最低语义门槛（SPIKE-003 结论）
- 不做草稿保存（DRAFT）—— 页面刷新即丢；MVP 极简
- 不做"最近导入的 5 集列表"—— 属于 Library / STORY-00060
- EpisodeCard "×" 删除按钮只清 session state，不调 DELETE /api/episodes/:id（那是 Library 功能）
