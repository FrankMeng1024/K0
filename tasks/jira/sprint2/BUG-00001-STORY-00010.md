# BUG-00001-STORY-00010: 返回按钮在直接URL访问/learn时触发GO_BACK未处理错误

**Priority**: Critical
**Sprint**: Sprint 2
**Opened by**: UX
**Status**: Verified

## Description
当用户直接通过 URL（如 `http://localhost:8081/learn`）访问 Learn 页面（无导航历史），点击 返回 按钮时，Expo Router 的 `router.back()` 抛出 "The action 'GO_BACK' was not handled by any navigator" 错误，在屏幕底部显示 toast 错误信息。

## Steps to Reproduce
1. 直接在浏览器地址栏输入 `http://localhost:8081/learn` 并访问
2. 点击 "← 首页" 返回按钮

## Expected
导航至 Home 页（`/`）

## Actual
toast 报错："The action 'GO_BACK' was not handled by any navigator"；页面无变化

## Root Cause
`app/learn.tsx` 使用 `router.back()`，当 navigation stack 为空时（直接 URL 访问），此调用抛出未处理异常。

## Fix
在 `app/learn.tsx` 中，将返回按钮改为：当有历史时 `router.back()`，否则 `router.replace('/')` 或使用 `router.canGoBack()` 判断。
Fix applied: `router.canGoBack() ? router.back() : router.replace('/')` — verified 0 errors on direct URL and normal navigation.
