# STORY-00901: 修 3 个 Blocker Bug — AppState 闭包 + 轮询并发 + 缺失监听

**Epic**: E-007 后台长任务无感回归
**Sprint**: Sprint 9
**Points**: 2
**Owner**: Frontend Dev
**Status**: Done

## Description
As a user, I want to switch away from the app while a URL is being processed and come back to see the correct progress immediately, so that the app never feels stuck.

## Expected Result
- 切走后回来，UI 立刻显示当前真实进度（服务器已跑到哪就是哪）
- 不会出现"卡在 20%"实际服务器已完成的状况
- Episode 屏也能在切走后回来自动刷新

## Acceptance Criteria
- [ ] `app/import/[jobId].tsx` 用 `useRef` 保存 `state` 供 AppState listener 读取，消除闭包过期
- [ ] 轮询加互斥锁（`pollingRef`），并发时忽略重复请求
- [ ] 前台恢复时清掉所有历史 timer，只留一个
- [ ] `app/episode/[id].tsx` 加 AppState listener，前台恢复时重新拉 pack
- [ ] Playwright 模拟：切换 tab → 回来 → 3 秒内进度追平服务器
- [ ] 零 console 错误
