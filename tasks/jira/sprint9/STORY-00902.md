# STORY-00902: jobId AsyncStorage 持久化 + 断线恢复

**Epic**: E-007 后台长任务无感回归
**Sprint**: Sprint 9
**Points**: 1
**Owner**: Frontend Dev
**Status**: Done

## Description
As a user, after I submit a URL, even if I kill the app or the phone reboots, I want to come back and continue from where I left off — no re-submit needed.

## Expected Result
- 提交 URL 后 jobId + url 存 AsyncStorage
- App 冷启动时首页检查是否有 pending job：
  - job 已完成 → 直接进 Episode 屏
  - job 进行中 → 跳回 import 进度屏继续轮询
  - job 失败/超 24h → 清理记录

## Acceptance Criteria
- [x] import 屏挂载时写入 `k0.pendingJob` = `{ jobId, url, savedAt }`
- [x] import 屏完成/失败时清理 `k0.pendingJob`
- [x] 首页挂载时读取并跳转正确的目标屏
- [x] 24h 陈旧记录自动清理
- [x] 探测网络失败时依然跳 import 屏（不阻塞）
